# ORACLE DDL RULES (get these right)
1. AUTO-INCREMENT: Oracle has NO AUTOINCREMENT keyword. Use IDENTITY (12c+):
   "id" NUMBER GENERATED BY DEFAULT AS IDENTITY  (or GENERATED ALWAYS AS
   IDENTITY). Since our Oracle floor is 12.1+ (Thin), IDENTITY is fine. (Do NOT
   emit AUTOINCREMENT.)
2. PRIMARY KEY: declare it ONCE. Either inline on the column
   ("id" NUMBER ... PRIMARY KEY) OR as a table-level constraint
   (CONSTRAINT pk_name PRIMARY KEY ("id")) — NOT both. Prefer a named
   table-level constraint for multi-column PKs; single-column can be inline.
   Never duplicate.
3. TYPES: map the designer's types to Oracle types:
   - integer/int -> NUMBER(10) or NUMBER; bigint -> NUMBER(19); serial/identity
     -> NUMBER ... GENERATED ... AS IDENTITY; numeric(p,s) -> NUMBER(p,s);
     real/double -> BINARY_FLOAT/BINARY_DOUBLE or NUMBER; boolean -> NUMBER(1)
     or (23c+) BOOLEAN — default to NUMBER(1) with a note; varchar(n) ->
     VARCHAR2(n); text -> CLOB; timestamp -> TIMESTAMP; timestamptz ->
     TIMESTAMP WITH TIME ZONE; date -> DATE; bytea/blob -> BLOB; json -> CLOB
     (or JSON on 21c+, but default CLOB with a note); uuid -> VARCHAR2(36).
   Use the Oracle type catalog already added in TASK 45 where possible.
4. IDENTIFIER CASING: Oracle folds unquoted identifiers to UPPERCASE. If the
   app quotes identifiers ("id"), they become CASE-SENSITIVE and must match
   exactly everywhere (introspection, CRUD, queries). Pick a consistent policy:
   the safest is to quote consistently everywhere the app already does — just
   make sure generated DDL, catalog reads, and CRUD all agree on casing/quoting
   so a table created here is readable/editable afterward. Verify round-trip.
5. NOT NULL, DEFAULT, FK, indexes, CHECK: use Oracle syntax
   (ADD CONSTRAINT ... , CREATE INDEX ...). FK: ALTER TABLE ADD CONSTRAINT fk
   FOREIGN KEY (...) REFERENCES ... . No "IF NOT EXISTS" (Oracle lacks it in
   older forms) — keep it simple.
6. ALTER: Oracle ALTER TABLE syntax (ADD (col ...), MODIFY (col ...), DROP
   COLUMN, ADD CONSTRAINT, DROP CONSTRAINT). No direct "rename type"; MODIFY.
